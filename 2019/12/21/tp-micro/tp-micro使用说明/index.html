    • api(接口管理)
      • pull_handler.gen.go(pull方式接口)

    • args(静态变量管理)

      • const.go(常量定义)
      • type.gen.go (自动生成结构体)
      • type.go(自定义结构体)
      • var.go(变量定义)
    • config(配置文件)
      • config.yaml
    • doc(项目文档管理)
      • APIDoc.md
      • databases.md
    • logic(业务层代码)
      • model
        • init.go(数据库控制)
      • tmp_code.gen.go(自动生成接口)
    • rerrs (错误处理管理)

    • config.go(项目初始化管理)

    二、模板文件结构说明

    type API_PULL interface

    • pull接口自动生成
    • pull接口即需要返回数据
    • 规范定义：
      • 定义接口以版本_模块_接口，如V1_Home_GetById
      • 生成对应接口路径即/v1/home/get_by_id
        type __API_PULL__ interface {
  V1_Home
}
type V1_Home interface {
  GetById(*GetByIdArgV1) *GetByIdResultV1
}
type (
  GetByIdArgV1 struct {
      id       int32
  }
  GetByIdResultV1 struct {
      id int32
      username string
  }
)

    type API_PUSH interface

    • push接口自动生成
    • push接口即没有返回数据
    • 规范定义：
        type __API_PUSH__ interface {
  // 订阅推送
      V1_Subscribe(*V1_SubscribeArg)
  }
  type V1_SubscribeArg struct {
      SessionId string `json:"session_id"` //uid
      Event     string `json:"event"`      //上下线状态
  }

    type MYSQL_MODEL struct

    • mysql自动生成Model文件
    • key:"pri"表示主键，key:"uni"表示唯一索引
    • 添加这两个表示，会自动生成相关操作的函数
    • 规范定义：
        type __MYSQL_MODEL__ struct {
      User
  }
  type User struct {
      Id   int64  `key:"pri"`
      Username string `key:"uni"`
  }

    type MONGO_MODEL struct

    • mongo自动生成Model文件
    • key:"pri"表示主键，key:"uni"表示唯一索引
    • 添加这两个表示，会自动生成相关操作的函数
    • 规范定义：
        type __MONGO_MODEL__ struct {
      UserMeta
  }
  type UserMeta struct {
      Id   int64  `key:"pri"`
      Username string `key:"uni"`
  }

    三、接口管理

    pull_handler.gen.go

    • pull方式接口(即有返回数据)
    • 功能：将api接口指向实际业务处理

    push_handler.gen.go

    • push方式接口(即不需要返回数据)
    • 功能：将api接口指向实际业务处理

    四、业务层

    tmp_code.gen.go

    • 模板文件自动生成的接口格式
    • 操作：复制到分类文件下，并删除或注释改接口

    五、内部客户端

    rpc.gen.go

    • 根据模板文件自动生成的客户端

    rpc.go

    • 自定义客户端接口
    • 如果需要对客户端参数进行修改，复制rpc.gen.go对应的接口，到该文件下进行修改即可

    六、项目初始化

    config.go

    1. type config struct
    • 读取配置文件config/config.yaml
      var cfg = &config{
  Srv: micro.SrvConfig{
      ListenAddress:     ":0",
      EnableHeartbeat:   true,
      PrintDetail:       true,
      CountTime:         true,
      SlowCometDuration: time.Millisecond * 500,
  },
  Etcd: etcd.EasyConfig{
      Endpoints: []string{"http://127.0.0.1:2379"},
  },
  Redis:       *redis.NewConfig(),
  CacheExpire: time.Hour * 24,
  LogLevel:    "TRACE",
}
    2. (c *config) Reload(bind cfgo.BindFunc)
    • 执行配置加载
    3. init()
    • 初始化配置

    七、Mysql逆向工程

    • 在数据库中建好数据库数据表，执行以下命令
      micro tpl -host [ip] -port [3306] -username qas -password [password] -db [DBName] -table [TableName] -ssh_user cds -ssh_host [跳板机地址] -ssh_port 38390
    • 命令执行后，会自动在模板文件中添加数据表结构生成结构，再执行micro gen生成model文件即可

    开发流程

    • 接口开发
      1. 编写模板文件
      2. 调整api
      3. 调整sdk
      4. logic实际业务逻辑开发
    • 读取配置文件
      1. 定义配置变量
      2. config.yaml配置变量内容
      3. 到config.go中读取文件配置，并在Reload赋值到变量
    • 接口插件控制
      1. 在api中针对接口或接口类添加插件拦截
      2. 如果有需要，在sdk对应接口添加插件例外条件
    • 调用其他项目接口
      1. 在config.go的Reload注册sdk
      2. 在具体业务位置调用sdk
